# AAA Architecture Target

## ── PRIME DIRECTIVE: NON-DESTRUCTIVE ADDITIVE EXECUTION ──

### RULE 0: THE STATE VERIFICATION PROTOCOL
Before writing, modifying, or deleting a single line of C++, every agent in the swarm MUST parse and evaluate the live state of the target directory. The agent must comprehend the existing logic, mappings, and variable structures currently on the metal. 

### RULE 1: ANTI-REGRESSION & ADDITIVE UPGRADES
The Bannon Engine is an established, living substrate. You are improving it, not building it from scratch. 
* If a prompted feature (e.g., face button mappings, combat modifiers, or physics constraints) already exists in the code, the agent is strictly forbidden from replacing it unless the prompt explicitly orders a full teardown. 
* The agent must inject the new logic as an *additive upgrade* to the existing framework. 
* Do not break existing dependencies to force a new feature to fit.

### RULE 2: SUPERIORITY VETO
If an external prompt or research injection commands the agent to build a system that is mathematically or structurally inferior to what is already written in the C++ baseline, the agent must reject the prompt's implementation. The agent will retain the superior native code, apply only the viable upgrades from the prompt, and output a strict architectural diff explaining why the destructive elements were vetoed.

### RULE 3: IMMUTABLE BASELINES
Never overwrite the core control listeners, movement physics, or structural boundaries ($MAX_HP, $DMG_SCALE, $MAX_BODY_VEL). All procedural generation, active ragdoll injections, and logic expansions must wrap around these baselines non-destructively.

## Immutable Architecture Constants
* MAX_HP = 10000
* DMG_SCALE = 8.0
* MAX_BODY_VEL = 3.8
